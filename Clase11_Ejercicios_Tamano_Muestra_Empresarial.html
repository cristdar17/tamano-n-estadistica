EJERCICIO 1

RapidEats — Tiempo promedio de entrega 🛵

📋 RapidEats es una startup de delivery en Pereira. El gerente de operaciones quiere estimar el tiempo promedio de entrega para incluirlo en la publicidad. Le pidió al equipo de datos que revisara los registros de la última semana como muestra piloto: analizaron 30 pedidos y encontraron que el tiempo promedio fue 28.4 minutos con una desviación estándar de s = 7.3 minutos. El gerente quiere que el margen de error no supere los 2 minutos, con 95% de confianza. ¿Cuántos pedidos deben muestrear?
1

Extraer datos del problema

s = 7.3 min, E = 2 min, Confianza 95% → Z = 1.960

Variable = media → población infinita (no nos dan N)

2

Elegir fórmula

n = (Z · s / E)²
3

Sustituir valores

n = (1.960 × 7.3 / 2)²
4

Resolver

n = (14.308 / 2)² = (7.154)² = 51.18
5

Redondear hacia arriba

n = 52 pedidos
🛵 Con solo 52 pedidos más (además de los 30 de la piloto), RapidEats puede decir en su publicidad: "Entregamos en 28 ± 2 minutos" con 95% de confianza.
EJERCICIO 2

BancoFlex — Migración a app móvil 📱

📋 BancoFlex tiene 12,000 clientes activos en su sucursal de Manizales. El área de transformación digital quiere estimar qué proporción de clientes estarían dispuestos a usar exclusivamente la app móvil (sin ir a oficina). Hicieron una encuesta piloto a 40 clientes: 26 dijeron que sí (p̂ = 0.65). Necesitan un error máximo de 4% con 95% de confianza. ¿A cuántos clientes deben encuestar?
1

Extraer datos

p̂ = 0.65, q̂ = 0.35, E = 0.04, Z = 1.960, N = 12,000

2

Fórmula (proporción, sin corrección)

n = Z² · p̂ · q̂ / E²
3

Sustituir y resolver

n = (1.960)² × 0.65 × 0.35 / (0.04)²
n = 3.8416 × 0.2275 / 0.0016
n = 0.87446 / 0.0016 = 546.54 → 547
4

Verificar corrección finita

547 / 12,000 = 0.0456 → 4.56% < 5% → NO necesita corrección
5

Resultado final

n = 547 clientes
⚠️ ¡Ojo! Aunque la población es finita (12,000), como n/N < 5%, la corrección no es necesaria. No siempre que N sea conocido hay que corregir.
EJERCICIO 3

CaféArte — Gasto promedio por cliente ☕

📋 CaféArte tiene 15 sucursales en el Eje Cafetero. La gerencia quiere estimar el gasto promedio por visita para rediseñar el menú. Tomaron datos del sistema POS de una semana (35 transacciones como piloto) y obtuvieron: promedio = $18,200 COP, desviación estándar s = $12,500 COP. La desviación es alta porque hay clientes que solo piden un tinto ($3,000) y otros que piden combos completos ($45,000). Quieren un error máximo de $2,000 COP con 95% de confianza.
1

Extraer datos

s = 12,500, E = 2,000, Z = 1.960, variable = media

2

Fórmula

n = (Z · s / E)²
3

Sustituir y resolver

n = (1.960 × 12,500 / 2,000)²
n = (24,500 / 2,000)² = (12.25)² = 150.06
4

Resultado

n = 151 transacciones
5

Observación importante

La alta variabilidad (s grande) infla n. Si quisieran E = $1,000 →

n = (1.960 × 12,500 / 1,000)² = (24.5)² = 600.25 → n = 601

¡Cuadruplica! Reducir el error a la mitad cuadruplica la muestra.

💸 Cuando la desviación estándar es grande relativa al error, n se dispara. En la práctica, el negocio debe decidir: ¿acepto más error o invierto en más datos?
EJERCICIO 4

ShopNow — Tasa de devolución 📦

📋 ShopNow es un e-commerce que acaba de lanzar una línea de ropa. El director financiero necesita estimar la tasa de devolución para calcular costos logísticos.

Escenario A: No tienen datos previos (producto nuevo).
Escenario B: Después de 2 semanas, tienen una muestra piloto de 60 ventas con 9 devoluciones (p̂ = 0.15).

En ambos casos: error máximo 3%, confianza 95%.
A

Escenario A — Sin piloto (p̂ = 0.5)

n = (1.960)² × 0.5 × 0.5 / (0.03)²
n = 3.8416 × 0.25 / 0.0009
n = 0.9604 / 0.0009 = 1,067.11 n = 1,068 ventas
B

Escenario B — Con piloto (p̂ = 0.15, q̂ = 0.85)

n = (1.960)² × 0.15 × 0.85 / (0.03)²
n = 3.8416 × 0.1275 / 0.0009
n = 0.48981 / 0.0009 = 544.22 n = 545 ventas
!

Comparación

¡La piloto ahorró 523 observaciones! (de 1,068 a 545)

EJERCICIO 5

ConectaPlus — Satisfacción del servicio 📡

📋 ConectaPlus tiene 45,000 usuarios de internet en Armenia. El área de experiencia del cliente quiere estimar la proporción de usuarios satisfechos. Una muestra piloto de 50 usuarios reveló que 34 están satisfechos (p̂ = 0.68). Requieren error máximo de 3% con 99% de confianza.
1

Extraer datos

p̂ = 0.68, q̂ = 0.32, E = 0.03, Z = 2.576, N = 45,000

2

Calcular sin corrección

n = (2.576)² × 0.68 × 0.32 / (0.03)²
n = 6.6358 × 0.2176 / 0.0009
n = 1.44395 / 0.0009 = 1,604.39 n = 1,605 usuarios
3

Verificar corrección finita

1,605 / 45,000 = 0.0357 → 3.57% < 5% → NO necesita corrección
4

Resultado final

n = 1,605 usuarios
📡 Aunque 45,000 parece finita, el n necesario (1,605) es solo 3.57% de la población. La corrección finita prácticamente no cambia nada.
EJERCICIO 6

TextilPro — Control de calidad por lote 🧵

📋 TextilPro produce lotes de 3,500 metros de tela. El jefe de calidad necesita estimar la proporción de metros con defectos visibles. En una inspección piloto de 45 metros, encontró 6 con defectos (p̂ = 0.133). Necesita error máximo de 2% con 99% de confianza (es un cliente exigente de exportación).
1

Extraer datos

N = 3,500, p̂ = 0.133, q̂ = 0.867, E = 0.02, Z = 2.576

2

Calcular sin corrección

n = (2.576)² × 0.133 × 0.867 / (0.02)²
n = 6.6358 × 0.11531 / 0.0004
n = 0.76533 / 0.0004 = 1,913.32 → 1,913
3

Verificar corrección finita

1,913 / 3,500 = 0.5466 → 54.7% > 5% → ¡SÍ necesita corrección!
4

Aplicar corrección por población finita

n_aj = n / (1 + (n−1)/N)
n_aj = 1,913 / (1 + 1,912/3,500)
n_aj = 1,913 / (1 + 0.5463)
n_aj = 1,913 / 1.5463 = 1,237.22 n = 1,238 metros
5

Ahorro

Se ahorraron 675 inspecciones (de 1,913 a 1,238)

⚠️ Cuando n/N es muy alto (como 54.7% aquí), la corrección hace una diferencia enorme. Sin ella, estarías inspeccionando 675 metros de más.
EJERCICIO 7

Hospital Santa Clara — Tiempo de espera ⏱️

📋 El Hospital Santa Clara de Dosquebradas atiende en promedio N = 200 pacientes por día en urgencias. La directora quiere estimar el tiempo promedio de espera para un informe a la Secretaría de Salud. Con una muestra piloto de 25 pacientes un martes, obtuvieron un tiempo promedio de 47 min con s = 18 min. Quiere error máximo de 5 minutos con 95% de confianza.
1

Extraer datos

N = 200, s = 18, E = 5, Z = 1.960, variable = media

2

Calcular sin corrección

n = (Z · s / E)² = (1.960 × 18 / 5)²
n = (35.28 / 5)² = (7.056)² = 49.79 → 50
3

Verificar corrección finita

50 / 200 = 0.25 → 25% > 5% → ¡SÍ necesita corrección!
4

Aplicar corrección

n_aj = n / (1 + (n−1)/N)
n_aj = 50 / (1 + 49/200)
n_aj = 50 / (1 + 0.245)
n_aj = 50 / 1.245 = 40.16 n = 41 pacientes
5

Ahorro

Se ahorraron 9 encuestas (de 50 a 41). En un hospital donde cada minuto cuenta, ¡eso importa!

EJERCICIO 8

GlobalTech — ¿Cuánto cuesta la precisión? 💰

📋 El VP de GlobalTech quiere estimar la proporción de empleados satisfechos con el programa de bienestar. Muestra piloto de 35 empleados: 24 satisfechos (p̂ = 0.686). Pide analizar: ¿cuántos empleados necesitamos encuestar con diferentes combinaciones de confianza (90%, 95%, 99%) y error (5%, 3%, 1%)? Población grande (multinacional, +50,000 empleados).

Fórmula: n = Z² × p̂ × q̂ / E²   donde p̂ · q̂ = 0.686 × 0.314 = 0.21540

Confianza Z E = 5% E = 3% E = 1%
90% 1.645 234 648 5,829
95% 1.960 332 920 8,276
99% 2.576 572 1,589 14,295
1

Caso más extremo: 99% confianza, 1% error

n = (2.576)² × 0.686 × 0.314 / (0.01)²
n = 6.635776 × 0.21540 / 0.0001
n = 1.42941 / 0.0001 = 14,294.1 n = 14,295 empleados
2

Caso más económico: 90% confianza, 5% error

n = (1.645)² × 0.686 × 0.314 / (0.05)²
n = 2.706025 × 0.21540 / 0.0025
n = 0.58290 / 0.0025 = 233.16 n = 234 empleados
💰 Pasar de E=3% a E=1% casi multiplica n por 10. El VP debe decidir: ¿vale la pena encuestar a 14,295 empleados en vez de 572?
Cuidado

Errores que se cometen en la práctica ⚠️

🔴

Confundir E en % vs decimal

Si el error es 4%, en la fórmula va 0.04, NO 4. Con E=4, n sale ridículamente pequeño. Trampa #1 en parciales.

🟡

Olvidar cuándo corregir por finita

No toda población finita necesita corrección. Solo cuando n/N > 5%. Una ciudad de 500,000 personas casi nunca necesita corrección.

🟣

Usar p̂ = 0.5 teniendo piloto

Si ya tienes una muestra piloto, ¡úsala! p̂ = 0.5 es el caso conservador para cuando NO tienes datos. Usarlo con datos disponibles desperdicia recursos.

🟢

No redondear hacia arriba

n = 384.02 NO es 384, es 385. Siempre se redondea al entero superior. Redondear hacia abajo reduce la confianza real por debajo del nivel deseado.

Lo que te llevas hoy 🎯

📖 De libro a empresa

Aprendiste a traducir un problema de negocio real en parámetros estadísticos: identificar si es media o proporción, extraer s o p̂, definir E según el contexto.

🧪 El poder de la piloto

Una muestra piloto de 30–50 datos puede ahorrarte cientos o miles de observaciones. Siempre pídela antes de calcular n a ciegas.

📐 Corrección finita

Solo cuando n/N > 5%. Aprendiste a verificar ANTES de aplicar, no aplicarla automáticamente.

📈 Sensibilidad

Menor error → n crece cuadráticamente. Mayor confianza → n crece. El negocio decide el trade-off entre precisión y costo.

🔥 La próxima vez que alguien en tu trabajo diga "encuestemos a 100 personas", tú ya sabes preguntar: ¿con qué confianza? ¿qué error aceptas? ¿tenemos datos previos? Eso te hace estadísticamente peligroso.
